Description

In real life, parking lots come in various sizes: from small ones by the townhouses to really large lots by malls or stations. At this stage, we will get better at "art imitating life" and make our program customizable by adding a  create command that allows us to specify the number of spots. For example, the command create 3 makes a parking lot with three spots. The number of spots should be positive. The program output should be the following: Created a parking lot with 3 spots.

The other commands like park or leave should return an error - Sorry, parking lot is not created. - until the user enters the create command. If the user calls create again, the previous parking state should be reset.

Also, it is important to keep track of which spots are occupied by which cars. For this, add a status command that prints all occupied spots in the ascending order. For each spot, it should print the spot number, the plate registration number and the color, all separated by spaces like in the example below. If there are no occupied spots, the program should print: Parking lot is empty..

Example

 The symbol > represents the user's input.

> park KA-01-HH-9999 White
Sorry, parking lot is not created.
> create 3
Created a parking lot with 3 spots.
> status
Parking lot is empty.
> park KA-01-HH-9999 White
White car parked on the spot 1.
> park KA-01-HH-3672 Green
Green car parked on the spot 2.
> park Rs-P-N-21 Red
Red car parked on the spot 3.
> leave 2
Spot 2 is free.
> status
1 KA-01-HH-9999 White
3 Rs-P-N-21 Red
> exit
